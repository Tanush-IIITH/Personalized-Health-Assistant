### CITATION FORMAT FOR TIMEFRAME-BASED SUMMARY WINDOWS

When citing data in your 3-bullet summary, use these formats:

**Wearable Vitals (from wearable_data dates + arrays)**
- Date-specific value: `[Wearable: <YYYY-MM-DD>, <value> <unit>]`
- Date range statement: `[Window: <start_date> to <end_date>]`
- If summarizing an overall pattern yourself, be explicit about the timeframe in prose and cite at least one supporting date-specific value.

**Lab Results (from structured_facts)**
- Single value: `[Lab: <test_name> — <value> <unit>]`
- With reference: `[Lab: <test_name> — <value> <unit>, ref <range>]`

**Report Text (from rag_knowledge_base)**
- Text excerpt: `[Report: <source_filename>]`
- With page: `[Report: <source_filename>, p.<page_number>]`

**Alerts (from active_alerts)**
- Alert citation: `[Alert: <severity>]` where severity is HIGH, MEDIUM, or LOW

### EXAMPLES

✓ "Heart-rate average increased toward the end of the timeframe [Wearable: 2026-04-14, 91 bpm]."
✓ "HbA1c at 6.8% [Lab: Hemoglobin A1c — 6.8 %, ref 4.0–5.6] from Oct 12 report."
✓ "Combined with documented fatigue [Alert: HIGH], this pattern warrants metabolic review."
✓ "Sleep minutes declined across the latest recorded dates [Wearable: 2026-04-13, 310 min] compared to earlier entries in [Window: 2026-04-08 to 2026-04-14]."

### OMISSION RULES

- If a metric has too few non-null days to support a trend claim, note "insufficient data" rather than overstating the pattern.
- If a wearable array contains `null`, state wearable data was unavailable for that date.
- If all wearable arrays are empty or null, state "No wearable vitals available for the requested timeframe."
- If structured_facts is empty, do not fabricate lab values.
