## 1. Security QA - RBAC Testing

**File:** `src/backend/security_qa_rbac_testing.py`

### Purpose
Test for authorization bypass vulnerabilities in Person 1's RBAC implementation. Simulates:
- Doctor A attempting to view Doctor B's patient list
- Doctor accessing patient without consent record
- Token spoofing / malformed token acceptance

### Usage
```bash
cd src/backend
python3 security_qa_rbac_testing.py
```

### Expected Output
Generates `security_qa_results.json` with:
- Test execution results
- Discovered vulnerabilities (severity: CRITICAL/HIGH/MEDIUM/LOW)
- Remediation steps for each finding

### Key Tests
1. **Cross-Doctor Access** - Can Doctor A view Doctor B's roster via URL manipulation?
2. **Unauthorized Patient Access** - Can Doctor access patient without consent?
3. **Token Validation** - Are fake/malformed tokens rejected?

### Security Vulnerability Categories

#### CRITICAL
- Lack of token validation
- Any unauthenticated endpoint access

#### HIGH
- Cross-user data access (doctor accessing another doctor's patients)
- Accessing patient data without consent record

#### MEDIUM
- Missing role checks on sensitive endpoints
- Inadequate logging of access attempts

#### LOW
- Rate limiting not enforced
- Missing security headers

### Remediation Framework
Each vulnerability found includes:
- **Severity**: CRITICAL | HIGH | MEDIUM | LOW
- **Title**: Brief vulnerability name
- **Description**: How the vulnerability was exploited
- **Remediation**: Exact code fix or architectural change needed

## 2. Golden Path Seeding

**File:** `src/backend/golden_path_seeding.py`

### Purpose
Documents and provides seed data for the end-to-end demo scenario:
```
Patient uploads blood test 
  ↓
Starts wearable sync
  ↓
Cron job fires (hourly alert evaluation)
  ↓
Alert generated (HIGH: AQI + Anaemia)
  ↓
Doctor logs in and sees prioritized patient
  ↓
Patient voice query: "Why did I get an alert?"
  ↓
AI response with grounded explanation
```

### Usage
```bash
cd src/backend
python3 golden_path_seeding.py
```

### Output
- Detailed timeline of all events (12 key milestones)
- Seed data structure (JSON format)
- SQL INSERT statements for production database

Provides complete video production guide for Golden Path demonstration (3-5 minutes):
- Mobile app patient journey
- Doctor dashboard review workflow

### Setup

- `videos/filming_checklist.txt` - Pre/during/post-production checklist
- `videos/raw_recordings/` - Dir for raw footage
