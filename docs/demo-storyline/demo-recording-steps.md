# Demo Recording Steps

This file is the practical checklist for recording the demo.

## Before You Record

1. Make sure the backend and any frontend/mobile builds are already running.
2. Log in once on the patient app and once on the doctor dashboard.
3. Preload the screens you need:
   - patient upload screen
   - patient report history
   - patient wearable/vitals screen
   - patient alert screen
   - patient assistant/voice screen
   - doctor patient list
   - doctor patient summary page
4. Turn on Do Not Disturb.
5. Close terminal windows that show errors or logs.
6. Keep only the required windows visible.

## Recommended Window Layout

### Option A: Alternate views

Best if you want the cleanest final video.

- record the mobile app flow first
- record the doctor dashboard flow second
- combine them during editing

### Option B: Side-by-side

Best if you want simultaneous comparison.

- left side: mobile emulator or phone mirror
- right side: doctor web dashboard in browser

Recommended screen ratio:
- mobile: 40%
- web dashboard: 60%

## Recording Sequence

1. Record the opening title card.
2. Record patient upload flow.
3. Record report processed/history screen.
4. Record wearable sync or vitals view.
5. Record alert appearing.
6. Record doctor patient list.
7. Record doctor summary view.
8. Record patient assistant question.
9. Record response.
10. Record end card.

## What To Say While Recording

Use [demo-video-script.md](/home/sasi-kumar/Downloads/ai.md/repo/docs/demo-storyline/demo-video-script.md).

If you are uncomfortable live-speaking while recording:

- record the screen silently first
- then record voice-over later

This usually produces a much smoother final demo.

## Editing Order

1. Put the opening title card first.
2. Insert patient upload clips.
3. Insert wearable clips.
4. Insert alert clip.
5. Insert doctor dashboard clips.
6. Insert voice assistant clip.
7. End with side-by-side comparison or final title card.

## Suggested On-Screen Text

Use short labels only:

- `Step 1: Upload Historical Report`
- `Step 2: Sync Wearable Data`
- `Step 3: Alert Generated`
- `Step 4: Doctor Reviews Summary`
- `Step 5: Patient Asks Why`

## Final Export

- Resolution: `1920x1080`
- Frame rate: `30 fps`
- Format: `MP4`
- Audio: clear voice-over, no background noise

## Final Check Before Submission

- no loading delays left in the edit
- patient name consistent throughout
- doctor name consistent throughout
- alert severity visible
- summary visible
- voice assistant question clearly readable
- final duration between 3 and 5 minutes
